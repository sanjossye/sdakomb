1. Konsep Dasar Binary Tree

Binary Tree adalah struktur data berbentuk pohon di mana setiap node memiliki maksimal dua anak: left child (kiri) dan right child (kanan). Jenis yang paling umum digunakan adalah Binary Search Tree (BST), di mana nilai node kiri selalu lebih kecil dari node induknya, dan nilai node kanan selalu lebih besar.

Dalam implementasi C++, setiap node direpresentasikan oleh sebuah struct TreeNode yang menyimpan data dan dua pointer ke anak kiri dan kanan. Seluruh operasi dikelola oleh class BinaryTree yang menyimpan pointer ke node akar (root).

Properti Binary Search Tree (BST):
  • Setiap node memiliki tepat satu nilai (data)
  • Semua node di subtree kiri memiliki nilai < nilai node saat ini
  • Semua node di subtree kanan memiliki nilai > nilai node saat ini
  • Setiap subtree kiri dan kanan juga merupakan BST yang valid
  • Tidak ada nilai duplikat (dalam implementasi standar)

Struktur Node

C++ 
struct TreeNode
{
    int data;         // Nilai yang disimpan
    TreeNode *left;   // Pointer ke anak kiri
    TreeNode *right;  // Pointer ke anak kanan

    // Constructor: inisialisasi node baru
    TreeNode(int value) : data(value), left(nullptr), right(nullptr) {}
};
Terminologi Penting:
  • Root: node paling atas, tidak memiliki induk
  • Leaf: node tanpa anak (left dan right = nullptr)
  • Height: jumlah edge terpanjang dari root ke leaf
  • Subtree: pohon yang terbentuk dari suatu node beserta semua turunannya

Contoh Visualisasi BST

Setelah insert: 40, 20, 60, 10, 30, 50, 70

           40          ← Root
          /  \
        20    60
       / \   / \
      10  30 50  70   ← Leaf nodes

Aturan: kiri < induk < kanan
  10 < 20 < 30  ✓
  20 < 40 < 60  ✓
  50 < 60 < 70  ✓

2. Insertion (Penyisipan Node)

Operasi insertion menambahkan node baru ke dalam BST sambil tetap mempertahankan properti BST. Proses dilakukan secara rekursif: bandingkan nilai baru dengan node saat ini, lalu arahkan ke kiri jika lebih kecil atau ke kanan jika lebih besar, sampai ditemukan posisi kosong (nullptr).

Kode Program

C++ 
TreeNode *insertNode(TreeNode *node, int value)
{
    // Base case: posisi kosong ditemukan, buat node baru
    if (node == nullptr)
    {
        return new TreeNode(value);
    }

    // Rekursi ke kiri jika value lebih kecil
    if (value < node->data)
    {
        node->left = insertNode(node->left, value);
    }
    // Rekursi ke kanan jika value lebih besar
    else
    {
        node->right = insertNode(node->right, value);
    }

    return node;
}

Penjelasan Program

Cara Kerja Insertion:
  • Jika node == nullptr → posisi yang tepat ditemukan, buat dan kembalikan node baru
  • Jika value < node->data → lanjutkan rekursi ke subtree kiri
  • Jika value > node->data → lanjutkan rekursi ke subtree kanan
  • Setiap pemanggilan rekursif mengembalikan pointer node yang sudah diperbarui
  • Node baru selalu disisipkan sebagai leaf node
⏱ Kompleksitas Insertion:
  • Best & Average Case: O(log n) — tree seimbang, setiap langkah memangkas setengah
  • Worst Case: O(n) — tree menjadi linear (seperti linked list) jika data dimasukkan berurutan

Contoh Output

tree.insert(40); → Root = 40
tree.insert(20); → 20 < 40, masuk ke kiri  → 40->left = 20
tree.insert(60); → 60 > 40, masuk ke kanan → 40->right = 60
tree.insert(10); → 10 < 40 → kiri, 10 < 20 → kiri → 20->left = 10
tree.insert(30); → 30 < 40 → kiri, 30 > 20 → kanan → 20->right = 30

3. Traversal (Penelusuran)

Traversal adalah proses mengunjungi setiap node dalam tree. Ada tiga urutan traversal utama, yang dibedakan oleh kapan node saat ini diproses relatif terhadap subtree kiri dan kanannya.

Preorder Traversal (Root → Left → Right)

C++ 
void preorderTraversal(TreeNode *node)
{
    if (node == nullptr) return;

    cout << node->data << " ";  // 1. Proses node saat ini (Root)
    preorderTraversal(node->left);   // 2. Kunjungi subtree kiri
    preorderTraversal(node->right);  // 3. Kunjungi subtree kanan
}

Inorder Traversal (Left → Root → Right)

C++ 
void inorderTraversal(TreeNode *node)
{
    if (node == nullptr) return;

    inorderTraversal(node->left);    // 1. Kunjungi subtree kiri
    cout << node->data << " ";  // 2. Proses node saat ini (Root)
    inorderTraversal(node->right);   // 3. Kunjungi subtree kanan
}

Postorder Traversal (Left → Right → Root)

C++ 
void postorderTraversal(TreeNode *node)
{
    if (node == nullptr) return;

    postorderTraversal(node->left);  // 1. Kunjungi subtree kiri
    postorderTraversal(node->right); // 2. Kunjungi subtree kanan
    cout << node->data << " ";  // 3. Proses node saat ini (Root)
}

Penjelasan Traversal

Perbedaan Ketiga Traversal:
  • Preorder — Root diproses sebelum anak-anaknya. Berguna untuk menyalin atau mencetak struktur tree.
  • Inorder — Root diproses di antara anak kiri dan kanan. Pada BST, inorder selalu menghasilkan urutan data yang terurut ascending.
  • Postorder — Root diproses setelah semua anaknya. Berguna untuk menghapus tree (hapus anak dulu, baru induk) atau mengevaluasi ekspresi.
⏱ Kompleksitas Traversal:
  • Semua traversal: O(n) — setiap node dikunjungi tepat satu kali
  • Space complexity: O(h) di mana h adalah tinggi tree (rekursi stack)

Contoh Output

BST yang digunakan:
           40
          /  \
        20    60
       / \   / \
      10  30 50  70

Preorder  (Root→L→R): 40 20 10 30 60 50 70
Inorder   (L→Root→R): 10 20 30 40 50 60 70  ← Selalu terurut!
Postorder (L→R→Root): 10 30 20 50 70 60 40

4. Deletion (Penghapusan Node)

Operasi deletion adalah yang paling kompleks karena harus mempertahankan properti BST setelah node dihapus. Ada tiga kasus yang harus ditangani berdasarkan jumlah anak dari node yang akan dihapus.

Kode Program

C++ 
TreeNode *findMinNode(TreeNode *node)
{
    // Cari node paling kiri (nilai terkecil di subtree kanan)
    while (node != nullptr && node->left != nullptr)
    {
        node = node->left;
    }
    return node;
}

TreeNode *removeNode(TreeNode *node, int value)
{
    if (node == nullptr)
        return node;

    // Cari node yang akan dihapus
    if (value < node->data)
        node->left = removeNode(node->left, value);
    else if (value > node->data)
        node->right = removeNode(node->right, value);
    else
    {
        // KASUS 1: Node tanpa anak (leaf) atau hanya satu anak
        if (node->left == nullptr)
        {
            TreeNode *temp = node->right;
            delete node;
            return temp;
        }
        else if (node->right == nullptr)
        {
            TreeNode *temp = node->left;
            delete node;
            return temp;
        }

        // KASUS 2: Node dengan dua anak
        // Ganti dengan inorder successor (nilai terkecil di subtree kanan)
        TreeNode *temp = findMinNode(node->right);
        node->data = temp->data;
        node->right = removeNode(node->right, temp->data);
    }

    return node;
}

Penjelasan Program

Tiga Kasus Deletion:
  • Kasus 1 — Node adalah leaf: Hapus langsung, kembalikan nullptr ke induknya.
  • Kasus 2 — Node punya satu anak: Hubungkan induk langsung ke cucu (bypass node yang dihapus), lalu delete node tersebut.
  • Kasus 3 — Node punya dua anak: Cari inorder successor (node terkecil di subtree kanan). Salin nilainya ke node saat ini, lalu hapus inorder successor dari subtree kanan.
💡 Mengapa Inorder Successor?
  • Inorder successor adalah nilai terkecil yang masih lebih besar dari node yang dihapus
  • Dengan menggantikan node dihapus menggunakan inorder successor, properti BST (kiri < root < kanan) tetap terjaga
  • Alternatif lain yang valid: gunakan inorder predecessor (nilai terbesar di subtree kiri)
⏱ Kompleksitas Deletion:
  • Best & Average Case: O(log n) — tree seimbang
  • Worst Case: O(n) — tree berbentuk linear

Contoh Output

BST awal:       40
               /  \
             20    60
            / \   / \
           10  30 50  70

Hapus node 20 (dua anak):
  → inorder successor dari 20 adalah 30
  → ganti data 20 dengan 30, hapus node 30 lama

Hasil:          40
               /  \
             30    60
            /     / \
           10     50  70

Hapus node 60 (dua anak):
  → inorder successor dari 60 adalah 70
  → ganti data 60 dengan 70, hapus node 70 lama

Hasil:          40
               /  \
             30    70
            /     /
           10     50

5. Kode Lengkap

Berikut adalah kode lengkap implementasi Binary Search Tree dalam C++, menggabungkan seluruh operasi: insertion, preorder/inorder/postorder traversal, dan deletion.

C++ 
#include <iostream>
using namespace std;

struct TreeNode
{
    int data;
    TreeNode *left;
    TreeNode *right;
    TreeNode(int value) : data(value), left(nullptr), right(nullptr) {}
};

class BinaryTree
{
private:
    TreeNode *root;

    TreeNode *insertNode(TreeNode *node, int value)
    {
        if (node == nullptr)
            return new TreeNode(value);
        if (value < node->data)
            node->left = insertNode(node->left, value);
        else
            node->right = insertNode(node->right, value);
        return node;
    }

    void preorderTraversal(TreeNode *node)
    {
        if (node == nullptr) return;
        cout << node->data << " ";
        preorderTraversal(node->left);
        preorderTraversal(node->right);
    }

    void inorderTraversal(TreeNode *node)
    {
        if (node == nullptr) return;
        inorderTraversal(node->left);
        cout << node->data << " ";
        inorderTraversal(node->right);
    }

    void postorderTraversal(TreeNode *node)
    {
        if (node == nullptr) return;
        postorderTraversal(node->left);
        postorderTraversal(node->right);
        cout << node->data << " ";
    }

    TreeNode *findMinNode(TreeNode *node)
    {
        while (node != nullptr && node->left != nullptr)
            node = node->left;
        return node;
    }

    TreeNode *removeNode(TreeNode *node, int value)
    {
        if (node == nullptr) return node;
        if (value < node->data)
            node->left = removeNode(node->left, value);
        else if (value > node->data)
            node->right = removeNode(node->right, value);
        else
        {
            if (node->left == nullptr)
            {
                TreeNode *temp = node->right;
                delete node; return temp;
            }
            else if (node->right == nullptr)
            {
                TreeNode *temp = node->left;
                delete node; return temp;
            }
            TreeNode *temp = findMinNode(node->right);
            node->data = temp->data;
            node->right = removeNode(node->right, temp->data);
        }
        return node;
    }

public:
    BinaryTree() : root(nullptr) {}

    void insert(int value) { root = insertNode(root, value); }

    void preorder()  { preorderTraversal(root);  cout << endl; }
    void inorder()   { inorderTraversal(root);   cout << endl; }
    void postorder() { postorderTraversal(root); cout << endl; }
    void remove(int value) { root = removeNode(root, value); }
};

int main()
{
    BinaryTree tree;

    tree.insert(40);
    tree.insert(20);
    tree.insert(60);
    tree.insert(10);
    tree.insert(30);
    tree.insert(50);
    tree.insert(70);

    cout << "Preorder Traversal:  ";
    tree.preorder();

    cout << "Inorder Traversal:   ";
    tree.inorder();

    cout << "Postorder Traversal: ";
    tree.postorder();

    tree.remove(20);
    cout << "\nSetelah hapus 20:" << endl;
    cout << "Inorder Traversal:   ";
    tree.inorder();

    return 0;
}

Output Program

Preorder Traversal:  40 20 10 30 60 50 70
Inorder Traversal:   10 20 30 40 50 60 70
Postorder Traversal: 10 30 20 50 70 60 40

Setelah hapus 20:
Inorder Traversal:   10 30 40 50 60 70

6. Ringkasan Operasi Binary Search Tree

Perbandingan Semua Operasi BST:
  • Insert → Rekursi kiri/kanan sampai posisi kosong. Best/Avg: O(log n), Worst: O(n)
  • Preorder → Root → Kiri → Kanan. Cocok untuk menyalin struktur tree. Selalu: O(n)
  • Inorder → Kiri → Root → Kanan. Menghasilkan data terurut ascending. Selalu: O(n)
  • Postorder → Kiri → Kanan → Root. Cocok untuk penghapusan tree. Selalu: O(n)
  • Delete → 3 kasus (leaf / 1 anak / 2 anak). Best/Avg: O(log n), Worst: O(n)
💡 Kapan BST Menjadi Tidak Efisien?
  • Jika data dimasukkan dalam urutan sudah terurut (misal: 10, 20, 30, 40...), tree menjadi seperti linked list — semua operasi menjadi O(n)
  • Solusi: gunakan Self-Balancing BST seperti AVL Tree atau Red-Black Tree yang menjamin O(log n) selalu
  • Untuk penggunaan sehari-hari: std::map dan std::set di C++ STL menggunakan Red-Black Tree secara internal

Kesimpulan

Binary Search Tree adalah struktur data yang sangat berguna untuk penyimpanan data terurut dengan operasi pencarian, penyisipan, dan penghapusan yang efisien. Kunci kekuatannya terletak pada properti BST: kiri < root < kanan, yang memungkinkan pencarian biner di setiap langkah.

Poin-Poin Penting:
  • Insertion: Selalu rekursif, node baru selalu jadi leaf. Pertahankan properti BST.
  • Inorder Traversal: Satu-satunya traversal yang menghasilkan data terurut di BST.
  • Deletion: Kasus dua anak adalah yang paling kompleks — gunakan inorder successor.
  • Semua operasi bergantung pada tinggi tree: tree seimbang = O(log n), tree linear = O(n).
  • Untuk data production, gunakan self-balancing tree (AVL, Red-Black) atau std::map/std::set.
